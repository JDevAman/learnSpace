# GOAL

Starter pack I'd suggest for anyone trying to build a career in Software Engineering (as a Sr. Engineer with 8 years across Amazon, Cars24 & now Nielsen)

1. Pick one primary language and build enough depth that you can debug memory, runtime, errors, packages, and edge cases without panicking.
2. Learn DSA through patterns like sliding window, binary search, graphs, DP, heaps, and trees to clear the DSA round, it's not dead yet.
3. Build 3 projects where you handle auth, database design, APIs, caching, background jobs, file uploads, error handling, and deployment. Pick a problem that annoys you the most, aim to make a better tool than what you already see.
4. Learn SQL because most products still face issues when queries, indexes, transactions, and schema design are poorly understood.
5. Get comfortable with Linux, Git, logs, networking basics, HTTP, DNS, ports, processes, and permissions because these show up everywhere in real work.
6. Learn system design by building small versions of systems like a URL shortener, payment flow, notification service, chat app, job queue, and search service.
7. Use AI like a senior assistant: ask it to compare approaches, generate edge cases, review your design, write tests, explain stack traces, and find failure paths.
8. Never paste AI code blindly into your project; make it explain every file, every dependency, every database change, and every assumption before you accept it.
9. Build the habit of reading codebases because production engineering is mostly improving existing systems, not writing fresh code from scratch every day.
10. Create proof of work in public through GitHub, short writeups, demos, architecture notes, and lessons learned so people can see how you think before they interview you.

Readme file for project:
a) problem statement
b) architecture diagram
c) database schema
d) API examples
e) tradeoffs
f) known limitations
g) what you would improve next

Debugging Stack:

- read the error properly
- check logs before guessing
- reproduce the issue
- isolate the failing layer
- inspect network calls
- check database state
- write down the root cause
